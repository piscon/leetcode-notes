# 0001. 两数之和 (Two Sum)

- **难度**：Easy
- **分类**：#数组 #哈希表

---

## 1. 题目原文

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target  的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

 

示例 1：

输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。

示例 2：

输入：nums = [3,2,4], target = 6
输出：[1,2]

示例 3：

输入：nums = [3,3], target = 6
输出：[0,1]

 

提示：

    2 <= nums.length <= 10的4次方
    -10的9次方 <= nums[i] <= 10的9次方
    -10的9次方 <= target <= 10的9次方
    只会存在一个有效答案

 

进阶：你可以想出一个时间复杂度小于 O(n2) 的算法吗？

---

## 2. 解题思路

这一题大概说的是，一个整数数组中可以找到两个数相加等于最终的目标整数值，找出这两个数在数组中的位置。已知限定条件可以避免该题无解，其他限制条件也在提示中。

### 初始思路：

从数组中的第一个数开始，依此往后相加；然后从数组第二个数开始，依此往后相加，直到得出解。

伪代码：

输入: nums (整数数组), target目标值

输出: 包含两个数字下标的数组

函数 twoSum(nums, target):外循环，遍历第一个数，从nums下标i=0开始，到i=nums最后第二位结束；内循环，从nums下标j=i+1开始，到nums最后一位结束。判断这两个数字相加是否等于目标值，即 nums[i] + nums[j] = target目标值。找到了，直接返回它们的下标[i, j]。

### 进阶思路：

创建一个数组（map），存入nums中的元素的数值(key)与下标(value)，依次存下。将存下的数与nums后面的元素遍历求解。

判断条件： target目标值- nums中元素的数值，是否存在于数组（map）中。如果不存在，就继续存入nums中第3、4个元素。如果能在数组（map）中找到解，就返回数组（map）的value以及当前nums元素的下标。

## 3. 复杂度分析

### 初始思路：
- **时间复杂度**：O(n^2) —— 原因：在最坏的情况下（即答案在数组的最末尾，或者根本没有答案时），计算机总共需要执行的相加比对次数是一个等差数列求和
- **空间复杂度**：O(1) —— 原因：不需要开辟额外的内存空间

### 进阶思路：
- **时间复杂度**：O(n) —— 原因：在最坏的情况下（即答案在数组的最末尾，或者根本没有答案时），计算机总共需要执行n-1次
- **空间复杂度**：O(n) —— 原因：最坏情况下，开辟N个内存空间
